# 产品需求文档 (PRD)：AI Builder - 智能正念陪伴捏捏乐 App

## 1. 项目概述
### 1.1 项目背景
现代社会生活节奏快，年轻人面临较大压力，市面上的“捏捏乐”多为单向发泄，容易滑向攻击性解压。本项目旨在将传统的解压玩具转化为支持双向触觉交互的正念修复装置，帮助用户觉察情绪、调节负面状态。

### 1.2 目标受众
面临较大工作或学习压力的年轻上班族和学生群体，他们寻求简单、便携的方式来缓解压力和焦虑，并看重产品带来的情绪价值。

### 1.3 核心价值定位
一个通过触觉感知帮助高压力群体识别、理解和重塑负面情绪的智能正念陪伴系统。

## 2. 核心功能说明

### 2.1 角色选取模块 (Role Selection)
- **功能描述**：用户在首次进入或设置中，可以选择一个陪伴自己进行正念呼吸和情绪调节的虚拟角色/数字生命。
- **交互细节**：展示多个角色的 2D/3D 形象和性格设定。用户选择后，该角色将在后续的引导和动画中作为主要形象出现。

### 2.2 实时行为检测与动画反馈 (Behavior Detection & Animation)
- **功能描述**：App 通过低功耗蓝牙 (BLE) 接收来自捏捏乐硬件（ESP32 + FSR 传感器）的实时压力和行为状态数据（如：拍打、急按、持续按压）。
- **交互细节**：
  - App 界面中心展示当前角色的动态动画。
  - 当接收到“消极行为”（如快速拍打、急促按压）时，动画表现为角色的担心、安抚或稍微晃动，并配以短文本（如“你现在有点急”）。
  - 当接收到“积极行为”（如持续稳定按压）时，动画表现为角色的舒展、放松和柔和微笑。

### 2.3 正念叙事引导 (Mindful Narrative Guidance)
- **功能描述**：系统识别到用户处于高唤醒状态（消极行为）时，App 会主动发起一段叙事化的正念引导。
- **交互细节**：
  - **Setup（建立场景）**：“我感受到你的触碰了”。
  - **Conflict（识别状态）**：“你的动作很快，是不是遇到什么烦心事了？”
  - **Guidance（推进转变）**：屏幕上出现呼吸节奏的视觉提示（如光圈的放大缩小），引导用户：“跟我一起，深呼吸，试着按住我三秒... 1... 2... 3...”。
  - **Resolution（确认变化）**：“现在比刚才稳定一些了，做得很好。”

### 2.4 情绪记录与报告 (Record & Report)
- **功能描述**：记录用户每次互动的数据，并生成可视化的情绪变化报告。
- **交互细节**：
  - **单次体验结论**：展示本次互动的主要行为（拍打/急促/持续）、状态判断（消极到积极的转化）以及是否成功完成正念修复。
  - **历史记录**：按天/周展示交互次数、各类行为（拍打、急促、稳定）的统计图表，以及“转化成功”的总次数。

## 3. 设计美学与体验方向
- **视觉风格**：温和、治愈、有机（Organic）。避免刺眼的颜色，主色调建议采用低饱和度的莫兰迪色系（如鼠尾草绿、柔和的米色、薰衣草紫）。
- **动效设计**：强调“呼吸感”。所有的动画过渡需要极其平滑，模拟真实的呼吸节奏。UI 元素的出现和消失应采用渐显渐隐，避免突兀的跳变。
- **排版排布**：大量的留白，使用圆润、具有亲和力的字体。界面不要过于拥挤，突出当前最重要的引导信息。

## 4. 技术依赖与约束
- **硬件通信**：App 必须支持 Web Bluetooth API (如果作为 PWA/Web App) 或原生 BLE (如果使用 React Native / Taro 打包为原生或小程序)。考虑到微信小程序在硬件互联上的普及率，建议后续可直接转为微信小程序架构。
- **响应延迟**：硬件数据到 App 动画的反馈延迟应控制在 200ms 以内，以保证触觉与视觉的同步感。
